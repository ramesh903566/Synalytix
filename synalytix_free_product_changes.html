Context: "Free" can mean two different things — know which one you chose. Fully free forever (no billing at all, open access) vs free tier (free plan with limits, paid plans later). The changes below cover both, with notes on what differs.
Backend changesCode changes required
Remove or bypass all feature gates
Every call to SubscriptionService.assertCapability() currently throws a 402 if the org is on the free plan. You need to either remove these checks entirely, or make the capability check always return true regardless of plan.
// src/services/SubscriptionService.ts // Option A — fully free, no gates at all async assertCapability(_orgId: string, _cap: string) { return; // always passes } // Option B — keep gates, unlock everything const ALL_CAPABILITIES = new Set([ 'nlq_queries', 'ai_insights', 'custom_dashboards', 'workflow_automation', 'api_access', 'advanced_exports' ]); async assertCapability(_orgId: string, cap: string) { if (!ALL_CAPABILITIES.has(cap)) throw new Error('Unknown capability'); return; // all caps unlocked }

Disable or remove Stripe billing routes
If fully free: remove /billing/checkout and /billing/portal routes entirely. Keep /billing/subscription so you can still show usage stats. Keep the webhook handler dormant — you may reactivate billing later.

Your plan config likely has hard limits on dataset rows, AI queries, seats, and API calls. Set these to a high number or remove the enforcement. Keep tracking usage — you'll want this data when you reintroduce pricing.

Strategic decisions to make nowThink through these
How long will it be free?
You need a clear answer — even internally. "Free during beta" with a rough timeline (3–6 months) is honest and sets expectations. "Free forever" is a bigger commitment that affects your entire business model. Pick one and communicate it clearly.
Decision required
Define your "free → paid" trigger now
Before you reintroduce pricing, you need data. Track: which features users love most, which orgs are most active, how much it costs you per active org (AI API + Supabase + Railway). That data shapes your pricing model. Start collecting it today.
Plan ahead
Set a cost ceiling per user
AI API calls (Anthropic) and Supabase storage cost real money. Calculate your max cost per active org per month. If a single org runs 10,000 NLQ queries, you're paying for that. Set a soft rate limit on AI endpoints to protect yourself — not as a paywall, but as abuse prevention.
// Keep this rate limit even on free plan: // AI endpoints: 200 req/day per org (soft cap) // Log when orgs hit 80% — reach out proactively // This is ops protection, not a pricing gate
Protect yourself
The right mindset: Going free is not giving up on revenue — it's buying user growth and usage data cheaply. Every active org is a future customer, a case study, or a referral. Treat them like gold.
